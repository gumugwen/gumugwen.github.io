    搭建Oracle运行环境

    一、项目的创建

    先用IDEA创建一个spring boot工程

    <build>
    <!--引入spring-boot-maven-plugin插件依赖 -->
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>

    二、项目的配置

    1、引入依赖

    （1）thymeleaf

    springboot整合thymeleaf，实现对静态资源的访问

    ①引入thymeleaf依赖

    <!--引入thymeleaf依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>

    ②html导入thymeleaf的名称空间

    <html lang="en" xmlns:th="http://www.thymeleaf.org">

    （2）mybatis

    ①引入mybatis的依赖，使用@Mapper注解

    <!--MyBatis -->
<dependency>
    <groupId>org.mybatis</groupId>
    <artifactId>mybatis</artifactId>
    <version>3.4.6</version>
</dependency>
<!--引入mybatis start-->
<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>2.1.3</version>
</dependency>

    ②配置application.yml

    Ⅰ：配置type-aliases-package

    mybatis:
  type-aliases-package: com.xzit.springboot.oracle.entity

    使用type-aliases-package中指定entity扫描包类让mybatis自定扫描到自定义的entity，配置完成后可以将原来的

    <resultMap id="TestResultMap" type="com.xzit.springboot.oracle.entity.Test">

    修改为

    <resultMap id="TestResultMap" type="Test">

    Ⅱ：扫描mybatis中的.xml文件

    mybatis:
  mapper-locations: classpath:mybatis/mapper/*.xml
  config-location: classpath:mybatis/mybatis_config.xml

    Ⅲ：配置mybatis_config.xml，控制台输出执行的sql语句

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <settings>
        <!-- 指定 MyBatis 所用日志的具体实现 -->
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>
</configuration>

    password:
    driver-class-name: oracle.jdbc.OracleDriver

        this.name = name;

    public int getAge() {

        <id column="test_id" property="id" jdbcType="VARCHAR" />
        <result column="test_name" property="name" jdbcType="VARCHAR" />
        <result column="test_age" property="age" jdbcType="BIGINT" />

        from t_test
    </sql>

    <!--查询所有Test数据 -->
    <select id="selectAll" resultMap="TestResultMap">
        <include refid="test_column" />
    </select>
</mapper>

    四、遇到的问题

    1、templates 文件夹下没有html，jsp或者ftl文件

    2020-11-17 15:05:31.235  WARN 3264 --- [           main] ion$DefaultTemplateResolverConfiguration : Cannot find template location: classpath:/templates/ (please add some templates or check your Thymeleaf configuration)

    解决方法：

    （1）在项目的 /src/main/resources/templates 目录下添加一个模板文件，比如 .jsp 或 .ftl 文件。

    （2）打开项目配置文件 application.properties，有的人可能喜欢使用 .yml 格式的。

    添加配置：

    ①application.properties

    spring.freemarker.checkTemplateLocation=false

    或者

    ②application.yml

    spring:
  thymeleaf:
    check-template-location: false

    2、IDEA Spring Boot中 自动注入mapper对象时出现红色下划线

    这个会出现下划线， 但是他不是报错，它是可以正常使用的，但是这对于强迫症患者来说，这会导致无法将项目进行下去啊

    解决办法：

    Autowiring for Bean class， 将原来的error级别改为warning即可

    3、ojdbc.jar包pom.xml出错

    Could not find artifact com.oracle:ojdbc6:pom:10.2.0.2.0 in alimaven (http://maven.aliyun.com/nexus/content/repositories/central/)

    解决方法：

    （1）如果有就进行第（2）步，因为缺少了jdbc驱动，需要下载ojdbc6.jar，下载完成后将其复制到Orac

    le的安装目录D:\app\Administrator\product\11.2.0\dbhome_1\jdbc\lib下（根据自己的安装目录找到jdbc\lib）

    （2）（需要先配置maven仓库）打开cmd窗口，执行命令（根据自己的ojdbc6.jar文件位置修改），出现BUILD SUCCESS就说明成功了

    mvn install:install-file -DgroupId=com.oracle -DartifactId=ojdbc6 -Dversion=10.2.0.2.0 -Dpackaging=jar -Dfile=D:\app\Administrator\product\11.2.0\dbhome_1\jdbc\lib\ojdbc6.jar

    （3）成功配置到了本地仓库

    （4）引入oracle依赖

    <!--引入oracle11g依赖 -->
<dependency>
    <groupId>com.oracle</groupId>
    <artifactId>ojdbc6</artifactId>
    <version>10.2.0.2.0</version>
</dependency>
