    一、Go语言打包

    1.1go程序打包

    linux目标平台得选对

    ::x86块
    set GOARCH=386

    ::x64块
    set GOARCH=amd64

    ::arm块
    set GOARCH=arm

    set GOARCH=386
    set GOOS=windows

    进入main.go所在目录

    主要是windows平台

    set CGO_ENABLED=0 //禁用CGO
set GOOS=linux //目标平台为linux
set GOARCH=amd64 //目标处理器架构是amd64
go build -o name //编译可执行文件到当前目录 （-o：自定义文件名）

    mac

    CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build

    2.linux上传运行

    • 查找指定端口的进程 博主的端口号9001

      netstat -tunlp|grep 5555

    • 结束之前的进程，开启新进程启动go服务

      • 4543是要结束的进程
      • blog2是二进制文件名
      • 使用&&同时执行2条命令，避免服务中断
      • 2>&1的意思 把标准错误（2）重定向到标准输出中（1），而标准输出又导入文件output里面，所以结果是标准错误和标准输出都导入文件output里面了
      • nohup+最后面的& 是让命令在后台执行
    //kill -9 4543 && nohup ./blog2 > nohup.log 2>&1 &

    kill -9 3384  && nohup ./blog2 &          //简单方式
