    docker自动化部署

    前提条件 docker镜像 tomcat + jenkins/jenkins

    安装

    docker pull tomcat
docker pull jenkins/jenkins:lts

    运行两个容器 tomcat01 和 myjenkins

    #运行docker 本地/home/docker_project和/usr/local/tomcat/webapps/my-project自动同步
docker run -it -v /home/docker_project:/usr/local/tomcat/webapps/my-project  --name tomcat01  -p 1016:8080 tomcat
#运行jenkins  外部端口8882访问 内部8080 50000是查看jenkins信息
#本地/home/docker_project和/var/tomcatspace自动同步
docker run -it -p 8882:8080 -p 50000:50000 -v /home/docker_project:/var/tomcatspace --name myjenkins jenkins/jenkins

    jenkins 拉取gitee项目 –>>linux本地目录 —>tomcat

    /var/tomcatspace /home/docker_project /usr/local/tomcat/webapps/my-project

    运行出现权限问题

    #解决权限问题
docker exec -u 0 -it id /bin/bash   #进入jenkins容器 的root用户
apt-get update
apt-get install sudo
apt-get install vim
root用户进入到Linux系统的/etc目录下
cd /etc
设置读写权限
chmod u+w /etc/sudoers
编辑sudoers文件，加入下面一句话 添加进最高权限组
jenkins ALL=(ALL)NOPASSWD:ALL

docker exec  -it id /bin/bash  #进入jenkins容器 的jenkins用户

    关于sudo无密码

    
#sudoers文件设置sudo命令无密码（root密码）登录
1. 当用户执行sudo时，Linux系统会去寻找/etc/sudoers文件，并且这是主动的，判断用户是否有执行sudo命令的权限
2. 如果用户有执行sudo的权限，让用户输入自己的密码来确认，这里比使用root的密码强多了，赞一下
3. 如果密码输入正确，开始执行sudo后面跟的shell命令
4. 如果要切换的身份相同， 和su命令一样，也不用输入密码

hadoop ALL=(root)NOPASSWD:ALL

这样也行

hadoop ALL=(ALL) ALL
